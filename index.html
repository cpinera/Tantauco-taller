I
Introducción

Bienvenida: IA & Agentes

La inteligencia artificial no es magia, es una herramienta. Imagina tener un asistente que entiende el lenguaje natural —como si le escribieras un mensaje— y que puede hacer tareas por ti: organizar información, responder preguntas, crear documentos o recordarte tus pendientes.

Un agente de IA va un paso más allá: es un asistente que no solo responde, sino que actúa. Puede conectarse a tus aplicaciones, leer tu lista de tareas, agregar nuevas y enviarte recordatorios, todo desde un chat de Telegram. Hoy vamos a construir uno juntos, desde cero, sin escribir una sola línea de código.

Todo lo que necesitas es un buen prompt —una instrucción bien escrita— y las herramientas correctas. Claude (la IA que usaremos) se encarga del resto.

El resultado final: una to-do list online que puedes gestionar desde Telegram, con un agente que entiende tus instrucciones en lenguaje natural.

III
Construcción · To-Do List

Construye tu To-Do List

Abre claude.ai y copia estos prompts en orden, todos en el mismo chat. Cada uno agrega una capa sobre lo anterior.

1
Prompt inicial · Crea la app
Quiero crear una página web con una lista de tareas. Las tareas deben poder clasificarse entre "por hacer", "en progreso" y "hecho", y los datos deben guardarse en el navegador para que no se pierdan al cerrar la página. Soy un usuario no programador — todas las instrucciones deben usar interfaces visuales, sin terminal ni línea de comandos. Empieza mostrándome un diagrama simple de la arquitectura completa antes de construir nada.
Claude mostrará primero el diagrama de arquitectura. Cuando lo apruebes, pasas al siguiente prompt.
2
Prompt de construcción · Genera el archivo
Apruebo la arquitectura. Ahora construye la aplicación completa y entrégame el código en un archivo descargable listo para usar. Nota: un solo archivo index.html. Datos en localStorage.
Claude generará el archivo index.html completo. Descárgalo — lo usarás en los pasos siguientes.
3
Prompt de deploy · Publica en GitHub y Vercel
Ahora dime paso a paso cómo subir este archivo a GitHub y publicarlo en Vercel para poder abrirlo en el navegador. Explica cada paso con mucho detalle — qué botones presionar, qué escribir en cada campo y en qué orden hacer todo. Nunca he usado GitHub ni Vercel antes. Nota: repositorio GitHub llamado todo-app. Sin README ni .gitignore.
Sigue las instrucciones que te dé Claude. Al final tendrás una URL pública con tu to-do list.
4
Prompt de mejora · Agrega prioridades
Quiero agregarle un sistema de prioridad del 1 al 10 a cada tarea, donde 10 es la más urgente y 1 la menos urgente. Las tareas más importantes deben aparecer primero en la lista mientras no estén completadas. Cada tarea debe mostrar visualmente su nivel de prioridad con un badge de color.
Úsalo en el mismo chat. Claude recuerda el contexto anterior y modifica la app existente.
5
✦ Momento aha
Prompt de diseño · Cambia el look & feel
Rediseña la interfaz completamente con una estética inspirada en Transformers: colores metálicos, azul energón, negro y gris oscuro, tipografía industrial y geométrica, bordes angulados. Que se sienta como un panel de control de Cybertron. Las columnas mantienen sus nombres originales: "Por hacer", "En progreso" y "Hecho". Nota: los cambios son exclusivamente estéticos — no modificar funcionalidad, datos ni estructura del código.
Observa cómo un solo prompt transforma completamente el aspecto visual. Este es el poder del diseño con IA.
6
Prompt de actualización · Sube la nueva versión
Tengo el código actualizado y necesito reemplazarlo en GitHub para que Vercel publique la nueva versión automáticamente. Dime paso a paso cómo hacerlo editando el archivo directamente en GitHub, sin subir uno nuevo. Nota: repositorio todo-app. Editar el archivo index.html existente, no subir uno nuevo.
Vercel detecta el cambio en GitHub y redespliega automáticamente en ~1 minuto.

1
Prompt de arquitectura · Chat nuevo en Claude
Quiero crear un bot de Telegram que funcione como asistente de mi to-do list. Debe poder agregar, editar y eliminar tareas con instrucciones en lenguaje natural. El to-do list está publicado en Vercel y actualmente guarda los datos en localStorage. El bot debe desplegarse en Railway. Soy un usuario no programador — todas las instrucciones deben usar interfaces visuales, sin terminal ni línea de comandos. Empieza mostrándome un diagrama de cómo se conectan todas las piezas antes de escribir cualquier código. Nota: el frontend debe migrar de localStorage a una API REST en Railway. La base de datos es PostgreSQL, compartida entre la API REST y el bot.
Ten a mano la URL de tu to-do list en Vercel. Claude mostrará primero el diagrama completo.
2
Prompt de construcción · Genera el código del bot
Apruebo la arquitectura. Entrégame el código del bot en archivos descargables. El bot debe usar Anthropic como motor de razonamiento para interpretar los mensajes del usuario en lenguaje natural y decidir qué acción ejecutar. Cuando tengas el código, dime paso a paso cómo crear un repositorio en GitHub y subir los archivos usando solo el navegador — qué botones presionar, qué nombres usar y en qué orden hacer todo. Nota: 3 archivos: requirements.txt, main.py, runtime.txt. Python 3.10 especificado en runtime.txt como "python-3.10". Librerías: python-telegram-bot==13.15, psycopg2-binary==2.9.9, flask==2.3.3, flask-cors==4.0.0, anthropic==0.40.0, httpx==0.27.2. PostgreSQL vía DATABASE_URL. CORS habilitado. Repositorio GitHub: todo-bot. Sin README ni .gitignore.
Claude generará los 3 archivos y te dirá cómo subirlos a GitHub.
3
Prompt de Telegram · Crea el bot con @BotFather
Dime paso a paso cómo crear mi bot en Telegram antes de subirlo a Railway. Explícame cada paso con detalle — qué aplicación abrir, qué escribir y dónde guardar el resultado, porque lo voy a necesitar en el siguiente paso. Nota: crear bot vía @BotFather en Telegram con /newbot. Guardar el token que entrega BotFather.
Guarda bien el token — lo necesitarás en el paso siguiente.
4
Prompt de deploy · Sube el bot a Railway
Dime paso a paso cómo subir el bot a Railway y dejarlo funcionando las 24 horas. Explícame qué secciones abrir, qué botones presionar y qué escribir en cada campo — quiero poder seguir las instrucciones sin saber nada de programación. Nota: start command: python main.py. Orden: proyecto vacío → PostgreSQL → agregar las 3 variables → repositorio todo-bot. Variables: DATABASE_URL (referencia Postgres.DATABASE_URL usando el botón "Add Reference", indicar el contenido exacto que quedará en el campo), TELEGRAM_TOKEN, ANTHROPIC_API_KEY.
Ten listos tu TELEGRAM_TOKEN y tu ANTHROPIC_API_KEY antes de empezar este paso.
5
Prompt de base de datos · Crea la tabla
Entrégame el código para crear la tabla de la base de datos en un archivo descargable, y dime exactamente dónde y cómo ejecutarlo dentro de Railway, paso a paso, usando solo botones y menús visuales. Nota: tabla tasks, columnas id (SERIAL PRIMARY KEY), text (TEXT NOT NULL), status (TEXT NOT NULL DEFAULT 'todo'), priority (INTEGER NOT NULL DEFAULT 5), created_at (TIMESTAMPTZ NOT NULL DEFAULT NOW()). Ejecutar en Railway → plugin PostgreSQL → pestaña Data → Query.
Este paso inicializa la base de datos. Sin esto el bot no puede guardar tareas.
6
Prompt de conexión · Obtén la URL de Railway
Ahora quiero conectar mi página web con el bot que acabo de subir. Dime paso a paso cómo encontrar la dirección del bot en Railway, cómo actualizar mi página para que use esa dirección, y cómo reemplazar el archivo en GitHub para que la página se actualice automáticamente. Nota: obtener URL pública en Railway → Settings → Networking → Generate Domain. Actualizar index.html reemplazando localStorage por llamadas a la API REST. Editar index.html en repositorio todo-app en GitHub para que Vercel redespliegue automáticamente.
Claude te dirá la URL exacta a buscar. Guárdala — la usarás en el siguiente prompt.
